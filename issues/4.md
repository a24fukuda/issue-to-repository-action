PR #3（マージ済み）で行った作業の詳細を記録として残します。

## きっかけ

issueの保存方法についての改善検討から始まり、特にコメントセクションの構造（`### @author — 日時` という見出しのみでコメントを区切る形式）に関心のある指摘があり、そこから機械可読性の改善、さらにリポジトリ全体の品質レビューへと発展しました。

## 1. コメント保存形式の変更

### 変更前の問題

- コメント本文が `### ` や `## Comments` を含むと、ファイルを機械的に読み戻す際にコメント境界が判別できない。
- コメント本文に閉じられていないコードフェンスがあると、1ファイルに連結された後続コメントのMarkdownレンダリングが崩れる（GitHub上ではコメント単位で独立レンダリングされるため起きない問題）。

### 実施した変更（`src/render.ts`）

- **引用ブロック化**: 各コメント本文の全行を1レベルの引用（`> `）にする。CommonMarkのコンテナブロック仕様により、内部で開かれたフェンスや見出しはコンテナ終端で強制的に閉じるため、後続コメントへのレンダリング崩れを構造的に防止。
- **HTMLコメントマーカー**: `<!-- issue-sync:comments -->` と `<!-- issue-sync:comment {"author":...,"created_at":...} -->` をコメントセクション・各コメント直前に埋め込み。Markdownプレビューでは非表示のまま、機械的に境界とメタデータを読み取れる。
- **CRLF正規化**: Issue本文・コメント本文の改行コード（`\r\n`・単独`\r`）をLFに正規化。GitHub APIがWebフォーム投稿の本文をCRLFで返すことによる、チェックアウト時の改行正規化との毎回の食い違い（無駄な再コミット）を解消。
- **参照パーサー**（`test/parse-helper.ts`）: render→parseのラウンドトリップで形式の機械可読性を保証するテスト専用実装。セクションマーカーの最終出現を採用する・`comments_count`と復元件数を照合するなど、本文由来のマーカー偽装やファイル破損を検出できるよう設計。

## 2. リポジトリ全体レビュー（1回目）

コメント形式の変更を機に、リポジトリ全体を6領域（`git.ts` / `github.ts`・`octokit.ts`・`concurrency.ts` / `run.ts`・`action.yml` / `sync.ts` / ワークフロー・セキュリティ / テストスイート）に分けてレビューし、24件の指摘（21件CONFIRMED、3件PLAUSIBLE）に対応しました。主な修正:

- **`src/concurrency.ts`**: 1件失敗した後もワーカーがバックグラウンドで残りアイテムのAPI呼び出しを続けていた問題（早期中断を追加）。
- **`src/sync.ts`**: シンボリックリンク経由でのリンク先上書き防止、破損マニフェストが古い所有ファイルを永久に削除不能にしていた問題の自己修復ロジック。
- **`src/git.ts`**: push失敗原因（拒否 vs 認証/ネットワーク/upstream未設定）の切り分け、committerのidentityを`git config`で永続化していた問題の修正。
- **`src/run.ts`**: `issues-dir`が未検証で、チェックアウト外への書き込み・削除が起き得た問題の修正。
- **ワークフロー**: `sync.yml`/`self-sync.yml`のcheckoutが古いSHAを固定してしまい、concurrencyで待機した実行が自分自身の前回コミットと衝突する問題への対処、`self-sync.yml`のトリガー拡張。
- **テスト**: 引数伝播・並行性下限・rejection伝播・push恒久失敗などの検証漏れを埋め、54→70件に増加。

## 3. 修正コミット自体のセルフレビュー（2回目）

1回目の修正コミット自体をさらにレビューし、修正の過程で新たに作り込んだ回帰4件（うち2件は実機再現込み）を発見・修正しました:

- **最重要（実機再現）**: `git config`の永続化をやめた際、push拒否後のfetch+rebaseリトライ経路にcommitter identityが渡らなくなり、実際のGitHub Actionsランナー相当の環境（identity未設定）で`git rebase`が失敗することを確認。`GIT_AUTHOR_*`/`GIT_COMMITTER_*`環境変数に統一して修正し、修正後に同一環境でリトライが成功することを再度実機検証。
- **実機再現**: issueファイル向けのシンボリックリンクガードが`.manifest.json`自体には未適用だった問題、`issues-dir`自体がシンボリックリンクでチェックアウト外を指すケースを検出できなかった問題（`realpath`ベースの検証に変更）。
- push拒否判定の文字列マッチングがロケール依存だった問題（`LANG=C`固定）、ログ文言の軽微な重複解消。

最終的に全72テストがパスし、CIも成功。PR #3としてマージ済みです。
